# 申请获取敏感数据

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/718)
> 分类: Getting Started


Shopee 开放平台会保护卖家的业务数据以及被视为敏感的用户个人数据（包括客户姓名、电话号码、电子邮箱地址和地址）。


默认情况下，敏感数据都会被脱敏处理。开发者需完成特定的安全合规要求，方可申请访问特定场景的未脱敏的敏感数据。

####**资格要求**

如需访问敏感业务数据，开发者须满足以下条件：

1. 提交渗透测试报告
 - 仅适用于**正在为或计划为泰国卖家提供服务的第三方合作伙伴平台（ISV）开发者**
 - 需通过 Open Platform Console 提交有效的渗透测试报告
1. 配置 IP 地址白名单
 - 适用于所有开发者
 - 托管您应用程序的服务器的 IP 地址必须声明并列入白名单。


⚠️ 注意：对于第三方合作平台（ISV）开发者，敏感数据访问权限有效期为 自渗透测试报告出具之日起**2 年**。

#### 如何提交渗透测试报告

请按照以下步骤上传您的渗透测试报告：

- 步骤 1：使用 developer account 登录 Open Platform Console。

*注意：member account 没有上传报告的权限。*

- 步骤 2：前往 Personal Center → Account Information （中国大陆 ISV：地址链接，其他地区 ISV：地址链接）
- 步骤 3：在“Security Reports & Certifications Information”下，单击“Add”。

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=3nkzhcjEysHs%2BxVIeQP%2BiarSKdEmhwKjczzYD%2FJArvxFyZzMjDJFfVJ8RFX6cNqLFAYlHxz94fUYl8ONlEFGHw%3D%3D&image_type=png)

- 步骤 4：在“Security Report & Certification Type”下，选择“Penetration Test Report”。
- 步骤 5：上传您最新的 Penetration Test Report
- 步骤 6：点击“Save”

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=ukOwqouxK6zyCz8GXhJf1UXdCDjwO%2F%2BPzT61qRe4vZHYPG6QovDt2Yd1bXTHPaxy3FKwqPXsb0870c350msmzQ%3D%3D&image_type=png)


**审核周期**

- 提交状态（通过 / 拒绝）将显示在「**Account Information**」页面
- 审核结果通常在**10 个工作日**内更新


📌 请参阅本页底部了解 渗透测试报告提交的**最佳实践和指南**。

#### 如何启用 IP 地址白名单

请按以下步骤为应用启用 IP 白名单：

- 步骤 1：登录您的 Open Platform 控制台并转到“App List”。
- 步骤 2：选择并点击应用
- 步骤三：点击“Go Live”并填写所需信息

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=%2FqddWrd%2FufgszAu0EqzuJupG12wDspo8uOp0rwhl4yXC3cHHIcNRI0w7zkUxisCvrxrLP%2BbA4iWbP9O81Ow%2FaA%3D%3D&image_type=png)

- 步骤 4：在“IP Address Whitelist”下，输入托管应用程序的服务器的 IP 地址。
- 步骤 5：将 Enable IP Address Whitelist 切换为 ON
- 步骤 6：点击 Submit

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=eOiVyGkoK1KQYE4yDiBl3WZ2X70hP9izMdRLceS%2BAx%2BYAbodKDHQRgvNjOHvrxGJejEGnIRDiOS9ln81sc3zZA%3D%3D&image_type=png)


⚠️**重要：**启用 IP 地址白名单后，API 调用只能从托管在已声明的 IP 地址上的应用程序发出。


#### 渗透测试报告提交的最佳实践

**1. 推荐测试机构**


为提升报告质量和审核效率，我们建议 ISV 选择信誉良好、具备资质的渗透测试机构：

-**所有 ISV：建议使用 CREST 认证的渗透测试机构**
-**中国大陆（CN）ISV：**建议选择以下机构：
 - Qianxin： Qianxin
 - 360： 360
 - 深信服:Sangfor
 - 长亭科技:Chaitin
- 说明：来自其他渗透测试机构的报告仍会被审核，但将以 case-by-case 的方式进行评估。


**2. 报告质量要求**


一份完整的渗透测试报告应当：

- 反映应用程序的外部暴露情况
- 评估所有相关系统或应用程序中的安全漏洞
- 包含完整的问题清单
- 明确说明不存在未解决的高危或严重风险问题


**3.建议报告发布日期**

- 建议 ISV 提交**近****1 年内******出具的渗透测试报告
- 敏感数据访问权限效期 为 自报告出具日起******2 年**
