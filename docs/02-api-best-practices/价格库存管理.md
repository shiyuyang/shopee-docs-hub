# 价格库存管理

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/223)
> 分类: API Best Practices

### 1.获取店铺商品价格
- 如果您的商品没有包含变体，请通过`v2.product.get_item_base_info` 接口获取商品价格；
- 如果您的商品包含变体，请通过 v2.product.get_model_list 接口获取商品价格。


下面是接口返回的价格信息样例：


    "price_info": [


                    {


                        "current_price": 7678,


                        "original_price": 13960,


                        "inflated_price_of_current_price": 9137,


                        "inflated_price_of_original_price": 16612,


                        "currency": "COP"


                    }


请注意：


1.如果您的商品参与了活动或者设置了折扣，current_price 将会在活动期间显示活动价格，非活动期间，则等于 original_price 原价。original_price 表示商品原价。


2.如果您的商品参与了多个活动，可以通过`v2.product.get_item_promotion`接口获取各种活动您设置的活动价格。


3.如果您是 ID / CO / PL 的卖家，请您注意 inflated_price_of_current_price/inflated_price_of_original_price 表示含税价；如果您是其他地区的卖家，inflated_price_of_current_price 字段仍然会返回，且 inflated_price_of_current_price=current_price，inflated_price_of_original_price 同理。

### 2.修改店铺商品价格

调用接口：`v2.product.update_price`

- 当您的商品包含多个变体，您可以在一次调用中，上传多个这个商品的多个变体价格进行修改。
- 这个接口只支持一次修改一个 item, 如果需要修改多个 item，请求多次即可。
- 价格支持修改的范围，请查看`v2.product.get_item_limit`接口中的 price_limit 字段


2.1修改没有变体的商品价格例子：


*{*


*"item_id": 1000,*


*"price_list": [{"original_price": 11.11}]*


*}*


2.2 修改包含变体的商品价格例子：


*{*


*"item_id": 2000,*


*"price_list": [{"model_id": 3456,"original_price": 11.11}，{"model_id": 1234,"original_price": 22.22}]*


*}*


注意，如果您是 ID / CO / PL 的卖家，修改价格（original_price）时，修改的是未含税价。

### 3.修改全球商品价格

*下列只适用升级了 CNSC/KRSC 的跨境卖家。


调用接口：`v2.global_product.update_price`

- 当您的全球商品包含多个变体时，您可以在一次调用中，修改这个全球商品的多个变体价格。
- 这个接口只支持一次修改一个 global_item_id, 如果需要修改多个 global_item_id,，请求多次即可。
- 全球商品的价格，请按照您设置的币种进行设置，可以通过`v2.merchant.get_merchant_info`接口进行查看币种设置。
- 如果您希望全球商品的价格经过 Shopee 计算后自动同步到店铺商品，请通过接口`v2.global_product.set_sync_field`设置价格同步，在更新全球商品价格后，将自动按照公式计算同步到店铺商品。如果您不希望通过全球商品管理店铺商品价格，请通过`v2.product.update_price`接口更新价格。

### 4.获取店铺商品库存
- 如果您的商品没有包含变体，请通过`v2.product.get_item_base_info` 接口获取商品库存；
- 如果您的商品包含变体，请通过 v2.product.get_model_list 接口获取商品库存。


下面是接口返回的库存信息样例：


```json
{                "stock_info_v2": {                    "summary_info": {                        "total_reserved_stock": 0,                        "total_available_stock": 389                    },                    "seller_stock": [                        {                            "location_id": "IDZ",                            "stock": 90                        }                    ],                    "shopee_stock": [                        {                            "location_id": "IDG",                            "stock": 99                        },                        {                            "location_id": "IDM",                            "stock": 200                        }                    ]                }            }
```


请注意：

- 在 stock_info_v2中，total_reserved_stock 表示所有活动预留库存，total_available_stock 表示所有可用库存，seller_stock 表示卖家库存，shopee_stock 表示 Shopee 仓库库存，一个商品可能同时存在 seller_stock 和 shopee_stock 库存，也可能会存在多个仓库的库存。
- 更多的库存计算逻辑，请参考 FAQ。

### 5.修改店铺商品库存

调用接口：`v2.product.update_stock`

- 当您的商品包含多个变体时，您可以在一次调用中，上传该商品的多个变体库存进行修改。
- 这个接口只支持一次修改一个 item, 如果需要修改多个 item，请求多次即可。
- normal_stock 字段即将弃用，请切换使用 seller_stock 字段进行修改库存。
- 卖家只能修改 seller_stock，不可以通过接口修改 shopee_stock。
- 库存支持修改的范围，请查看`v2.product.get_item_limit`接口中的 stock_limit 字段。


5.1 修改没有变体的商品库存例子：


{


"item_id": 1000,


"stock_list": [{"seller_stock": [{"stock": 100}]}]


}


5.2 修改包含变体的商品库存例子：


{


"item_id": 2000,


"stock_list": [{"model_id": 3456,"seller_stock": [{"stock": 100}]}，{"model_id": 1234,"seller_stock": [{"stock": 100}]}]


}


请注意：

- 如果商品包含变体，变体之间的价格差不能超过一定的倍数
| 地区 | 倍数 |
| --- | --- |
| BR | 4 |
| SG/VN/TW/TH/PH/MX | 5 |
| ID/MY | 7 |
| CL/CO | 9 |
| CNSC | 7 |

- 商品参与某些活动，禁止修改商品原价，详细查看 FAQ: https://open.shopee.com/faq/140
### 6.修改全球商品库存

*下列只适用升级了 CNSC/KRSC 的跨境卖家。


调用接口：`v2.global_product.update_stock`

- 当您的全球商品包含多个变体时，您可以在一次调用中，修改该全球商品的多个变体库存。
- 由于升级了 CNSC/KRSC 的跨境卖家，只能通过全球商品管理店铺商品库存，因此您只能通过`v2.global_product.update_stock`接口管理店铺商品库存，使用`v2.product.update_stock`接口修改库存将报错。
- 这个接口只支持一次修改一个 global_item_id, 如果需要修改多个 global_item_id，请求多次即可。
- 库存支持修改的范围，请查看`v2.global_product.get_global_item_limit`接口中的 stock_limit 字段。
