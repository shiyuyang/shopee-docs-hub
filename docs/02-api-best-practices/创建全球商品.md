# 创建全球商品

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/213)
> 分类: API Best Practices


**本文章适用于升级了 CNSC/KRSC 的跨境卖家*

## 全球商品是什么？

全球商品是为跨境卖家提出，您可以通过它一站式管理多个市场店铺的商品。它的优势体现在以下方面：

- 语言自动翻译：卖家可以用中文或英文填写全球商品的商品名称和描述，并设置同步更新到多个店铺商品，Shopee 将会自动翻译成当地语言。
- 库存统一管理：卖家可以通过全球商品统一管理多个商品的库存。
- 商品信息同步更新：对于一些无市场差异化的商品信息，卖家可以通过修改全球商品，即可同步更新到多个店铺商品的商品信息。
- 商品一键定价：Shopee 提供定价公式和公式因子的配置，卖家可以通过更新全球商品的价格，并设置同步更新到多个店铺商品，Shopee 将会根据公式计算的结果更新到店铺商品。


目前升级了 CNSC 的中国跨境卖家和升级了 KRSC 的韩国跨境卖家，必须使用全球商品管理商品。对于此类卖家，请注意商品管理的流程将会和本土卖家存在差异，您需要另外对接 Merchant API 和 Global Product API 接口。对于其他例如订单相关的接口，无需做流程上的修改。

## 初识 CNSC/KRSC 账号体系

下面我们将讲解升级了 CNSC 或者 KRSC 账号体系相关概念。（只适用跨境卖家）

术语说明：

- Main account(主账户)：又称母账户，权限最高的卖家账户，账户拥有者通常是公司法人，主账户可以管理该卖家的所有商店和所有子账户。主账户的登录格式为 XXX:main，例如 maidehao:main。
- Merchant（商家）：即主账户名下的公司，一个主账户激活后可以添加多个公司。
- Shop（商店）：即主账户名下的商店。一个 Merchant 下可以存在多个市场的店铺。主账户可以将商店授权给不同的子账户管理。


1）升级了 CNSC/KRSC 的店铺进行授权时，务必使用主账号登录。如何进行主账号授权请参考文章《授权与鉴权》。


2）一个主账号下，可以有多个 Merchant，每个 merchant 下可以有多个市场的 shop，包括 SIP 的店铺，而且，相同市场也可以存在多个 shop，例如图中 Merchant 1下同时有 SG shop1 和 SG shop 2。

## 商品管理流程变化

采用全球商品管理后，对于商品管理的流程，请注意以下几点的改变：

### 1. 创建商品

原流程还需要通过`v2.product.add_item`接口即可完成商品创建，现在必须通过`v2.global_product.add_global_item`接口创建全球商品，再调用`v2.global_product.create_publish_task`接口将全球商品发布到各个市场，生成店铺商品，具体发布流程请看文章《发布全球商品》。

### 2. 字段管理限制

我们将商品相关的字段分成三类：


1.只能通过 MTSKU 管理字段：这些字段无市场差异，例如类目，属性等，卖家通过更新 MTSKU 的字段值，Shopee 将会同步更新到 MPSKU；


2.MTSKU 和 MPSKU 共同管理字段：这些字段可以有市场差异化，例如商品名称和商品描述，卖家可以选择通过 MTSKU 的字段值进行更新，则 Shopee 会经过翻译后同步更新到 MPSKU，或者卖家选择单独更新某个市场的 MPSKU 字段值，并不修改 MTSKU，则更新只生效于该 MPSKU。


3.只能通过 MPSKU 管理字段：这些字段不通过全球商品管理，例如 MPSKU 的物流。

| 商品字段 | MTSKU 管理 | MPSKU 管理 |
| --- | --- | --- |
| 类目（category_id) | ✓ | × |
| 属性（attribute) | ✓ | × |
| 品牌（brand) | ✓ | × |
| 商品名称（item_name) | ✓ | ✓ |
| 商品描述（description） | ✓ | ✓ |
| 商品重量（weight) | ✓ | × |
| 商品尺寸（dimension) | ✓ | × |
| 商品价格（price） | ✓ | ✓ |
|**商品库存（stock)**| ✓ | × |
| 商品图片和视频（image/video) | ✓ | ✓ |
| 商品发货天数（days_to_ship) | ✓ | ✓ |
|**商品状态（item_status)**| ✓ | ✓ |
| 商品条件（condition) | ✓ | × |
| 商品 SKU (item sku) | ✓ | × |
| 商品变体数量（tier_variation 结构) | ✓ | × |
| 商品变体规格名称和变体名称（name/option) | ✓ | ✓ |
| 商品物流 | × | ✓ |


请注意：

- 库存逻辑：假设全球商品 MTSKU 库存为10，且 MTSKU 已发布店铺商品 MPSKU1和 MPSKU2，则 MPSKU1库存=MPSKU2库存=10,当买家购买了一个 MPSKU1 时，即 MPSKU1=9 ，则 MTSKU= MPSKU2=9。
- 如果全球商品被删除，则全球商品发布过的所有店铺商品也将会被删除。但是店铺商品可以单独删除和下架而不影响全球商品。
### 3. 商品基本数据差异

如果您初次对接 Shopee 商品管理，建议先通读《商品创建准备》文章，因为创建全球商品和创建普通商品的字段基本相同。下面我们只讲述创建全球商品调用的接口和注意点。

#### 3.1 获取全球商品类目

调用接口：`v2.global_product.get_category`


请注意：

- 对于不同的 merchant, 这个接口返回的类目树数据一样。
- V2.global_product.get_category API 返回创建全球商品可用的类目，但是可能会出现这样一种情况，类目 A 在马来西亚店铺不可以销售，在巴西店铺可以销售，我们将会在发布全球商品过程中提示您不可将全球商品发布到马来西亚，如果您想精确获取到各个市场的可用类目，可以调用`v2.product.get_category`
#### 3.2 获取全球商品推荐类目

调用接口：`v2.global_product.category_recommend`

#### 3.3 获取全球商品类目属性

调用接口：`v2.global_product.get_attribute_tree`


请注意：

-**need_region_mandatory**请求参数**和 mandatory_region**响应参数的作用：因为对于某些属性，比如 NCC 属性，只有台湾市场的产品是强制的，如果卖家不想发布到台湾商店，我们不想拦截全球产品的正常创建，所以这个属性对于全球产品来说是可选的，但是我们会通过“mandatory_region”响应参数提示这些特定属性所需的市场代码。
- 除了第一点所说的特殊属性以外，其他属性，只要有一个市场要求必填，则 `v2.global_product.get_attribute_tree` 返回必填。
#### 3.4 获取全球商品推荐属性

调用接口：`v2.global_product.get_recommend_attribute`

#### 3.5 获取全球商品品牌列表

调用接口：`v2.global_product.get_brand_list`

#### 3.6 获取全球商品发货天数

调用接口：`v2.global_product.get_dts_limit`


请注意：


V2.global_product.get_dts_limit 将会返回所有市场能填写发货天数的并集，即返回的情况可以为


 "days_to_ship_range_list": [


       {


                 "min_limit": 7,


                "max_limit": 8


           },


          {


              "min_limit": 10,


                "max_limit": 13


           } ]


此时，全球商品可用的发货天数是7，8，10，11，12，13。


您也可以保存`v2.product.get_dts_limit`的接口数据，获得各个市场更加精确的发货天数要求。

#### 3.7 获取全球商品类目是否支持尺寸表数据

调用接口：`v2.global_product.support_size_chart`


请注意：


我们正在推出表格类型的尺寸表格式，详情请看公告。

#### 3.8 获取全球商品数据限制

调用接口：`v2.global_product.get_global_item_limit`


请注意：


**text_length_multiplier**的作用，由于 MTSKU 商品名称/描述/变体规格名称和选项名称，支持输入中文和英文，而中文语义密度较大，即用相同字符长度的中文翻译比用英文翻译成市场语言，信息会更长，所以为了中文和英文有不同的字符长度限制，我们新增了**text_length_multiplier**字段表示中文字符的计算倍数。即当接口返回：


 "text_length_multiplier": 2.44,


  "global_item_name_length_limit": {


            "min_limit": 1,


            "max_limit": 500


},


表示卖家如果用英文填写产品名称，则最多填写1～500个字符；如果卖家用中文填写产品名称，则最多填写1～204个字符 （500/2.44约等于204.92），描述/变体规格名称和选项名称同理。

### 4.创建全球商品流程
#### 4.1 卖家完成 CNSC/KRSC 基础配置

卖家需要登录 CNSC/KRSC 完成币种和汇率等配置后，才能进行创建全球商品，否则 v2.global_product.add_global_item 接口将会报错


- CNSC 卖家了解更多操作指引
 - 设置基础货币：https://shopee.cn/edu/article/10614
 - 设置市场汇率：https://shopee.cn/edu/article/3904
- KRSC 卖家请点击这里了解更多操作指引
#### 4.2 完成主账号授权

详细请参考《授权与鉴权》文章中，主账号授权部分

#### 4.3 获取主账号下所有的 merchant id list 和 shop id list

可以调用下列接口：


第一步：授权后通过`v2.public.get_access_token` 获取到当前授权成功的 merchant id list 和 shop id list。


第二步：调用`v2.merchant.get_merchant_info` 接口获取每个已授权 merchant 的信息。


第三步：调用`v2.merchant.get_shop_list_by_merchant`接口获取每个 merchant 下已授权的 shop 列表。


第四步：调用`v2.shop.get_shop_info`接口获取每个已授权 shop 的信息，包括 SIP 主店铺和附属店铺的关系。

- 如果是普通店铺，is_sip= false 且 sip_affi_shops 字段不返回；
- 如果是 SIP 主店铺，is_sip=true 且 sip_affi_shops 字段返回；
- 如果是 SIP 附属店铺 is_sip=true 且 sip_affi_shops 字段不返回。

#### 4.4 设置商品信息同步开关

卖家可以通过 CNSC/KRSC 或者接口设置全球商品信息的更新是否自动同步到店铺商品。

#### 4.5 创建全球商品


4.5.1上传媒体文件


商品的图片（image）和视频（video），都需要**预先**存储在 Shopee 的媒体资料库中，所以您需要先调用 media_space 相关的接口上传。详细可参考文章《创建商品》 “上传媒体文件”部分。


上传图片和视频后，您就可以调用`v2.global_product.add_global_item`接口创建商品。


4.5.2上传类目和类目属性


1）上传类目


全球商品必须上传类目（category_id）。您只能选择`v2.global_product.get_category`接口中返回“has_children”=false 的 ID 进行上传。


2）上传属性


全球商品必须上传必填的属性（attribute_list）。您可以通过《获取商品基础信息》文章了解到各种属性类型。各种属性类型如何上传，详细可参考文章《创建商品》 “上传属性”部分


4.5.3上传全球商品名称


全球商品名称**（global_item_name**）只能填写纯简体中文或纯英文和数字。请不要混合输入中英文，如“漂亮牌 dress ready stock”，这样可能会导致部分标题没有被翻译为当地语言，导致商品被禁卖；


4.5.4上传全球商品描述


全球商品描述（description​​）只能填写简体中文、英文和数字。


*****请注意：


目前我们支持白名单卖家可以使用带图文的商品描述（extended_description）,如何使用详细参考**FAQ****。**


4.5.5上传全球商品价格


全球商品的价格（original_price），中国跨境卖家可以使用人民币或者美元进行定价，韩国跨境卖家可以使用美元定价，中国跨境卖家设置的币种，您需要查看`v2.merchant.get_merchant_info`接口中的 merchant_currency 字段，根据卖家选择的币种上传对应币种的价格。


4.5.6上传全球商品库存


请注意：


例如 MTSKU 一共发布了三个市场，店铺商品分别是 MPSKU1, MPSKU2,MPSKU3, 则 MTSKU 库存设置为100时，库存 MPSKU1=MPSKU2=MPSKU3=100,当 MPSKU1售出1个商品后，库存 MTSKU=MPSKU1=MPSKU2=MPSKU3=99


4.6 创建变体


当您创建商品成功之后，`v2.global_product.add_global_item`将会返回 global_item_id，这是全球商品的唯一标识。如果您还需要定义规格，创建多个选项变体，例如商品的颜色尺寸，可以调用`v2.global_product.init_tier_variation`接口创建变体，调用成功后，调用`v2.global_product.get_global_model_list` API 获取每个变体的 global_model_id，作为变体的唯一标识。


4.6.1 创建一层规格变体


假设商品有尺寸的规格，尺寸包含 XS、S 和 M。我们称这种商品是1-tier variation 的结构。

| tier_index | size | price | stock | sku |
| --- | --- | --- | --- | --- |
| tier_index[0] | XS | 100 | 10 | sku1 |
| tier_index[1] | S | 200 | 20 | sku2 |
| tier_index[2] | M | 300 | 30 | sku3 |


`v2.global_product.init_tier_variation`请求参数如下：


```json
{    "global_item_id": 800188562,    "tier_variation": [        {            "name": "size",            "option_list": [                {                    "option": "XS",                    "image": {"image_id":"82becb4830bd2ee90ad6acf8a9dc26d7"}                },                {                    "option": "S",        "image": {"image_id":"72becb4830bd2ee90ad6acf879dc26d7"}                },                {                    "option": "M", "image": {"image_id":"92becb4830bd2ee90ad6acf8a9dc26d7"}                }            ]        }    ],    "model": [        {            "tier_index": [0],            "original_price": 100,            "global_model_sku": "sku1",            "normal_stock": 10        },        {            "tier_index": [1],            "original_price": 200,            "global_model_sku": "sku1",            "normal_stock": 20        },       {            "tier_index": [2],            "original_price": 300,            "global_model_sku": "sku3",            "normal_stock": 30        }    ]}
```


4.6.2 创建两层规格变体


假设商品有颜色和尺寸的规格，颜色包括红色和蓝色，尺寸包含 XL 和 L。我们称这种商品是2-tier variation 的结构，因此此例子中最多有2*2=4 个变体。

| tier_index | color | size | price | stock | sku |
| --- | --- | --- | --- | --- | --- |
| tier_index[0,0] | Red | XL | 100 | 10 | sku1 |
| tier_index[0,1] | Red | L | 200 | 20 | sku2 |
| tier_index[1,0] | Blue | XL | 300 | 30 | sku3 |
| tier_index[1,1] | Blue | L | 400 | 40 | sku4 |


`v2.global_product.init_tier_variation`请求参数如下：


```json
 {  "global_item_id": 100917481,  "tier_variation": [    {      "name": "color",      "option_list": [        {          "image": {"image_id": "82becb4830bd2ee90ad6acf8a9dc26d7"},          "option": "Red"        },        {          "image": {"image_id": "72becb4830bd2ee90ad6acf879dc26d7"},          "option": "Blue"        }      ]    },    {      "name": "size",      "option_list": [        {          "option": "XL"        },        {          "option": "L"        }      ]    }  ],  "model": [    {      "tier_index": [0,0],      "original_price": 100,      "normal_stock": 10,      "global_model_sku": "sku1"    },    {     "tier_index": [0,1],      "original_price": 200,      "normal_stock": 20,      "global_model_sku": "sku2"    },    {      "tier_index": [1,0],      "original_price": 300,      "normal_stock": 30,      "global_model_sku": "sku3"    },    {     "tier_index": [1,1],      "original_price": 400,      "normal_stock": 40,      "global_model_sku": "sku4"    }  ]}
```


请注意：

- 您定义的 option，我们会按顺序显示 Shopee 商城端，Shopee 目前只支持定义最多两层规格。
- 您可以为每个变体定义一张图片。如果是两层规格的商品，您只能定义第一层 option，也就是例子中您可以根据颜色定义变体图片，不能根据尺寸定义变体图片。一旦要添加变体图片，则第一层所有的 option 都需要定义图片。我们不允许部分 option 有图片，部分 option 没有图片。图片需要先调用`v2.media_space.upload_image`接口上传。
- 请注意您定义的 tier_index 必须从0开始定义且不能溢出，否则会报错。
- 建议您创建商品后间隔5秒后创建变体，因为我们底层创建商品数据可能存在延迟。
- 创建成功后您可以调用`v2.global_product.get_global_model_list`接口获取到每个 tier_index 对应的 model id。


下一步请查看如何发布全球商品
