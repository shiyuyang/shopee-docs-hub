# 发布全球商品

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/215)
> 分类: API Best Practices


**本文章适用于升级了 CNSC/KRSC 的跨境卖家*


创建完全球商品之后，您可以将全球商品发布到各个市场，但如果全球商品已经在某个市场发布过了对应的店铺商品，则不能再发布到这个市场，即一个全球商品在一个市场只能存在一个店铺商品。


下面是流程讲解：

## 第一步：获取可发布店铺列表

调用接口：`v2.global_product.get_publishable_shop`


您可以通过接口获取到全球商品可以发布的店铺列表，但是对于下面的情况，我们将不会返回对应的店铺

- 没有完成店铺授权的店铺
- SIP 的附属店铺
- 已经发布过的市场店铺
## 第二步： 获取店铺渠道

调用接口：`v2.logistics.get_channel_list`


您需要调用`v2.logistics.get_channel_list`接口并选择**enable=true 且 mask_channel_id=0**的渠道，为店铺商品设置渠道。如果您发布全球商品时不上传店铺渠道，我们将默认帮您选择已开启且可用的店铺渠道。

## 第三步：发布全球商品

调用接口：`v2.global_product.create_publish_task`


您在发布商品时，非必填字段，如果您不上传，则默认 Shopee 处理后用于店铺商品。如果您上传，则为自定义值，Shopee 不会做任何处理。具体逻辑如下：

| 字段名 | 上传自定义的值 |
| --- | --- |
|**item_name**|**YES**: 卖家需要翻译当地语言后上传**NO**: Shopee 将帮助翻译成当地语言 |
|**description_info**/**description**|**YES**: 卖家需要翻译当地语言后上传**NO**: Shopee 将帮助翻译成当地语言 |
|**original_price**|**YES**: 卖家需要上传当地币种的价格**NO**: Shopee 根据全球商品和计算公式计算得到当地价格 |
|**tier_variation--name**|**YES**: 卖家需要翻译当地语言后上传**NO**: Shopee 将帮助翻译成当地语言 |
|**tier_variation--option_list--option**|**YES**: 卖家需要翻译当地语言后上传**NO**: Shopee 将帮助翻译成当地语言 |
|**image**|**YES**: 卖家上传自定义图片**NO**: Shopee 将采用全球商品的图片 |



接口调用成功后，我们将会返回一个发布任务 id: publish_task_id。


调用接口：`v2.global_product.get_publish_task_result`


此接口将返回发布任务成功与否。如果成功，将会返回发布成功的 item_id、shop_id 和 region 信息，如果失败将会返回具体的失败原因。请注意：如果发布的店铺是 SIP 主店铺，则发布成功之后，Shopee 会自动发布此全球商品到这个主店铺下的附属店铺。

## 第四步：获取已发布店铺商品列表

调用接口：`v2.global_product.get_published_list`


调用此接口，将会返回该全球商品所有历史已经成功发布过的店铺商品 item_id 和 shop_id, 包含 Shopee 自动发布到附属店铺的店铺商品。请注意：此接口不返回未完成授权且已发布完成的店铺商品。
