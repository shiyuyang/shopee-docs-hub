# 商品信息管理

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/221)
> 分类: API Best Practices

## 1. 获取商品列表
### 1.1 获取所有店铺商品

**调用接口：**<u>**v2.product.get_item_list**</u>



这个接口支持您获取店铺所有商品ID列表，或者通过更新时间范围和状态进行筛选。


### 1.2 根据特定条件搜索商品

**调用接口：**<u>**v2.product.search_item**</u>



这个接口支持您根据一些特定条件搜索商品ID列表，条件包括：

- 包含商品名称关键词的商品ID列表
- 包含sku关键词的商品ID列表
- 缺乏必填属性的商品ID列表
- 缺乏选填属性的商品ID列表
## 2. 获取商品基础信息

**调用接口：**<u>**v2.product.get_item_base_info**</u>** + **<u>**v2.product.get_model_list**</u>



如果商品不包含变体，您只需要调用<u>v2.product.get_item_base_info</u>即可获取到商品基础信息和商品库存价格信息，否则还需要同时调用<u>v2.product.get_model_list</u>接口获取变体的库存价格信息，<u>v2.product.get_item_base_info</u> API 中的“has_model”字段表示产品是否有变体。


## 3. 获取商品数据

**调用接口：**<u>**v2.product.get_item_extra_info**</u>



这个接口可以获取商品浏览量、点赞数、销量、评价数量、评价星级数据。


浏览量数据为最近30天统计值，销量数据为累计值。


## 4. 获取商品活动信息

**调用接口：**<u>**v2.product.get_item_promotion**</u>



这个接口可以获取到商品参与的所有进行中或即将开始的活动信息。如果商品同时参与了多种活动，<u>v2.product.get_item_base_info</u> 的promotion_id字段将返回其中一种活动ID，建议您继续调用接口<u>v2.product.get_item_promotion</u>拿完整的活动信息数据。

## 5. 更新商品信息

**调用接口：**<u>**v2.product.update_item**</u>



1.这个接口支持更新除了size_chart/price/stock/变体的商品信息。被上传字段都将会被更新，不被上传的字段将不更新。



2.对于**item_sku**/**wholesale**/**video_upload_id** 我们支持删除操作，您可以上传空字符串即可实现删除。



删除item_sku的请求参数例子：


{


    "item_id": 800182459,


    "item_sku": ""


}



3.更新富文本商品描述注意事项，请参考<u>FAQ</u>。



4.如果您在更新商品时并没有上传某些字段，但是遇到了提示这些字段填写错误，这种情况是正常的，因为每次您的更新，我们都会全量校验商品信息合法性，所以如果不符合要求，请您修改。

## 6. 商品上下架和删除

**调用接口：**<u>**v2.product.unlist_item**</u>



如果传unlist=true, 表示将商品下架，如果传unlist=false,表示将商品重新上架。



**调用接口：**<u>**v2.product.delete_item**</u>



这个接口可以将商品状态改为删除，请注意，删除商品后，将不可以再进行商品更新操作，而且卖家也无法通过卖家中心查看到。对于Shopee deleted 和Seller deleted的商品，在90天内，您仍然可以通过接口获取到商品信息，在90天之后，商品数据将在数据库中永久删除，您无法查询到此商品的任何信息，如果您有需要，请及时保存商品信息。

## 7. 更新商品尺寸表图片

**调用接口：**<u>**v2.product.update_size_chart**</u>



这个接口可以新增或者更新商品的图片尺寸表，如果遇到报错“Your shop can not edit image size chart“，请查看<u>公告</u>。

## 8. 注册品牌

**调用接口：**<u>**v2.product.register_brand**</u>



您可以通过这个接口注册自有品牌，Shopee审核成功您将会获得一个有效的brand_id 用于新增或更新商品。


具体审核流程可以查看<u>FAQ</u>。




**下面内容仅适用于CNSC/KRSC卖家。*

## 9. 获取全球商品列表

**调用接口：**<u>**v2.global_product.get_global_item_list**</u>



这个接口支持您获取对应Merchant下所有的全球商品ID列表或者通过更新时间范围。接口将不支持返回被删除的全球商品ID列表。

## 10. 获取全球商品ID

**调用接口：**<u>**v2.global_product.get_global_item_id**</u>



通过这个接口，您可以快速找到店铺商品对应的全球商品ID。

## 11. 获取全球商品信息

**调用接口：**<u>**v2.global_product.get_global_item_info**</u>** + **<u>**v2.global_product.get_global_model_list**</u>



1.如果全球商品不包含变体，您只需要调用<u>v2.global_product.get_global_item_info</u>即可获取到全球商品基础信息和库存价格信息。如果商品包含变体，您还需要调用<u>v2.global_product.get_global_model_list</u>接口获取到变体的库存价格信息。



2.店铺商品数据和活动信息不会通过全球商品维护，您可以调用<u>v2.product.get_item_extra_info</u>获取店铺商品数据，调用<u>v2.product.get_item_promotion</u>获取店铺商品参与的活动信息。

## 12. 更新全球商品

**调用接口：**<u>**v2.global_product.update_global_item**</u>



由于店铺商品的部分字段可以通过全球商品和店铺商品共同管理，详细字段可以查看《<u>创建全球商品</u>》文章，因此通过接口<u>v2.global_product.set_sync_field</u>设置字段同步，在更新全球商品信息后，Shopee将自动同步到店铺商品。

## 13. 删除全球商品

**调用接口：**<u>**v2.global_product.delete_global_item**</u>



全球商品不支持下架，只能删除，全球商品删除后，被发布的所有店铺商品也将会被删除。

## 14. 更新全球商品尺寸表图片

**调用接口：**<u>**v2.global_product.update_size_chart**</u>



这个接口可以新增或者更新全球商品的图片尺寸表，如果您想单独更新某店铺商品的尺寸表，也可以调用<u>v2.product.update_size_chart</u>接口。




*您可以查阅这篇<u>FAQ</u>了解更多CNSC/ KRSC 的卖家拥有哪些product模块的接口权限。
