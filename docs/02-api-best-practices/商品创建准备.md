# 商品创建准备

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/209)
> 分类: API Best Practices


创建商品前，您需要通过 API 拿到创建商品所需要的类目，属性，品牌，发货天数，类目是否支持尺寸表等信息，为创建商品做准备。

## 1. 类目

每个商品都有唯一的一个类目，您可以通过`v2.product.get_category`接口获取 Shopee 支持的所有类目数据。每一个节点的类目都有唯一的 category_id。

### 1.1 全球类目数据

为了方便卖家销售多个市场，Shopee 的类目树数据适用于所有的市场，但我们仍然会根据市场的地方政策，将部分类目设置为市场禁售，即会出现`v2.product.get_category`接口用马来西亚店铺请求，您可以获取到类目 A，但是通过新加坡店铺，您获取不到类目 A 的情况。这是因为不同站点可用的类目不同。跨境和本土销售的政策也有所不同，所以类目数据也会有差异。对于不同类型的卖家，支持的类目数据也有细小的差异。


所以，为了您拿到最准确类目数据，建议根据不同店铺进行获取。 

### 1.2 类目树

V2.product.get_category 接口返回该店铺所有可用的类目。


parent_category_id=0时，表示这是第一级类目，否则，将返回这个类目的上一级类目，当​​​​has_children=false 表示最后一级类目，否则，表示这个类目还有子类目。请注意，创建或者更新商品只能使用 has_children=false 的类目 ID。

### 1.3 推荐类目

为了帮助卖家快速找到商品类目，您也可以调用​​​​​​​​`v2.product.category_recommend`接口，接口将根据商品名称和商品图片，返回推荐的类目列表。

## 2. 属性

每个类目有不同的属性数据。`v2.product.get_attribute_tree` 接口将返回某个类目的属性数据，但是请注意只能查询最后一级类目的属性数据。

### 2.1 必填属性和可选属性

当您创建或者更新商品时，商品必须具有所有必填属性的值，但可以缺少可选属性。`v2.product.get_attribute_tree` 接口返回参数​​​​​​​​​​is_mandatory：true 表示必填属性，false 为可选属性。

### 2.2 属性的类型

根据属性值是否支持多选、是否支持自定义填写、是否提供可选属性值，我们将属性区分了多种类型，您可以通过 `v2.product.get_attribute_tree` 接口返回参数​​input_type 获取得到：


| 属性类型 | 输入类型 | 是否支持自定义 | 是否提供可选属性值 |
| --- | --- | --- | --- |
|**DROP_DOWN**| 单选下拉选择 | 否 | 是 |
|**MULTIPLE_SELECT**| 多选下拉选择 | 否 | 是 |
|**TEXT_FILED**| 文本框 | 是 | 否 |
|**COMBO_BOX**| 单选下拉选择+文本框 | 是 | 是 |
|**MULTIPLE_SELECT_COMBO_BOX**| 多选下拉选择+文本框 | 是 | 是 |

### 2.3 属性值的数据类型

属性值有规定的数据类型，您需要按照要求的数据类型上传属性值。您可以通过`v2.product.get_attribute_tree` 接口返回参数 input_validation_type 参数获取属性值的数据类型。数据类型包括：


-**INT_TYPE**：整型
-**STRING_TYPE**：字符串类型
-**ENUM_TYPE**：枚举值类型
-**FLOAT_TYPE**：浮点类型
-**TIMESTAMP_TYPE**：时间戳类型，例如1624007700
-**DATE_TYPE**：日期类型，包含两种，一种年月日，一种年月，例如06/2021 或者31/06/2021，您可以通过 `v2.product.get_attribute_tree` 接口返回参数 date_format_type 获取到日期类型。

请注意：实际创建/更新商品时的 attribute value 请输入时间戳，但是获取商品详情 attribute value 将会返回日期类型。

### 2.4 属性值单位

例如，长度这个属性，我们允许卖家选择“cm”、“m”等单位，所以您需要知道哪些属性的属性值可以填写单位以及属性值可用哪些单位。您可以调用`v2.product.get_attribute_tree` 接口获取这些信息。


返回参数**format_type**：QUANTITATIVE 表示属性值要填写单位，NORMAL 表示属性值不可以填写单位。


返回参数**attribute_unit**返回这个属性的属性值可用的单位。


*对于 Shopee 提供的属性值列表中，我们也会将属性值的数值和单位分开显示，即 attribute_value_list 中的 original_value_name 将返回属性值的数值，value_unit 将返回属性值的单位。

### 2.6 推荐属性

为了帮助卖家快速找到商品推荐填写的属性，您也可以调用​​​​​​​​`v2.product.get_recommend_attribute`接口，卖家可以上传商品名称和商品图片，接口将会返回推荐的属性列表。注意，这个接口可能不会返回必填的属性。

## 3. 品牌

您可以为商品上传品牌信息，但我们会要求某些类目需要必须填写品牌信息，并且每个类目支持填写的品牌列表不同，所以您可以通过`v2.product.get_brand_list`接口将获取到某个类目（只能用最后一级类目请求）支持的品牌列表数据。


参数 is_mandatory：true 表示这个类目必须填写品牌信息。


*如果 Shopee 提供的品牌列表不包含您所需要的品牌，您可以向 Shopee 提交注册品牌申请，请调用`v2.product.register_brand` API，审核详细流程请参考 FAQ。


*如果您的商品没有品牌，Shopee 提供的品牌列表中，包含 No Brand（brand_id: 0)的选项，您可以选择此选项进行上传。


`v2.product.get_brand_list` 接口请求参数中，status：1为 Shopee 提供的品牌列表，2为您提交申请后等待审核的品牌列表。

## ​​4. 发货天数

我们对不同的类目和不同类型的卖家，都有不同的发货天数要求，卖家需要在发货天数内履行订单，对于特定的类目，我们支持您设置商品为预售商品，发货天数将比较长。您可以通过 `v2.product.get_item_limit` 接口获取发货天数信息。


如果 days_to_ship_limit 里 min_limit 和 max_limit 同时返回的值是-1，则表示类目不支持预售，当返回大于0的值时，即为这个类目可以填写的预售天数范围。如果类目不支持预售，non_pre_order_days_to_ship 参数将返回 Shopee 为这个类目设置的商品发货天数。

## 5. 尺寸表

我们支持部分类目的商品上传尺寸表，您可以通过`v2.product.support_size_chart`接口查询到类目（最后一级类目）是否支持图片类型的尺寸表。


*请注意，我们正在推出表格类型的尺寸表格式，目前只有部分白名单卖家可以通过卖家中心使用，open api 还不支持返回支持表格类型的类目以及上传表格类型的尺寸表，所以白名单卖家请忽略通过`v2.product.support_size_chart`接口获取的结果。

## 6. 商品信息限制

我们对商品信息有一定的限制，例如商品名称能填写的字符长度，商品价格填写范围等，我们针对不同的市场，不同类型的卖家都有不同的限制。您可以通过`v2.product.get_item_limit` 接口获取到我们设置的范围。

## 7. 商品物流渠道

商品能选的物流渠道您可以通过`v2.logistics.get_channel_list`接口获取 logistics_channel_id​​，您只能选择为商品选择**enabled=true 且 mask_channel_id=0**的渠道。
