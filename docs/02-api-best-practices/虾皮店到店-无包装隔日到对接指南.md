# 虾皮店到店 - 无包装隔日到 对接指南

> 来源: [Shopee Open Platform](https://open.shopee.com/developer-guide/677)
> 分类: API Best Practices

###**1. 名词解释**
- 虾皮店到店 - 无包装隔日到：物流渠道 ID 30029
- 虾皮商品编码（unpackage sku id）：用于该物流选项的商品唯一识别码
- 虾皮出货标签（unpackage shipment label）：使用该物流选项发货时，需贴在商品上的标签
- 装箱单（TO label）：使用该物流选项发货时，需贴在外箱上的标签
- 拣货分区（sorting group）：用于该物流选项商品装箱及发货时的分区
###**2. 物流选项运作模式**

该物流选项相较于其他选项，具有以下特殊运作模式：

####**2-1 自动拆分订单**

针对选择该物流选项的订单，当订单状态变为 “READY_TO_SHIP” 时，系统将根据商品数量（Quantity）自动将订单拆分为多个包裹（package）。例如：一笔订单包含 3 瓶奶茶、2 瓶红茶，系统会自动将该订单拆分为 5 个包裹（package），每个包裹（package）中包含 1 件商品。

####**2-2 订单发货方式**

选择该物流选项的订单发货时，无需按订单维度包装商品，只需在要号后完成以下步骤

1. 在每件商品上贴好虾皮出货标签（unpackage shipment label）；
1. 按拣货分区（sorting group）将商品装箱，并在外箱上贴好装箱单（TO label），即可完成发货。


详细信息可参考卖家帮助中心文章：【蝦皮店到店-無包裝隔日到】出貨流程教學

###**3. 获取虾皮商品编码（unpackage_sku_id）**

虾皮商品编码（unpackage_sku_id）是该物流选项的商品唯一识别码。当商品开启 “虾皮店到店 - 无包装隔日到” 物流选项后，可通过以下 API 获取相关信息：

- v2.product.search_unpackaged_model_list
###**4. 获取订单信息或明细**

请注意，选择该物流选项的订单会自动拆分，需确保获取包裹（package）层级的信息，避免遗漏订单数据。

- v2.order.search_package_list：获取不同包裹（package）状态的包裹列表
- v2.order.get_package_detail：获取包裹（package）明细信息
- v2.order.get_order_detail：获取订单明细信息，调用时务必在请求参数（request）的 “response_optional_fields” 中加入 “package_list”

* 当该物流选项订单的包裹物流状态（PackageFulfillmentStatus）流转至 “LOGISTICS_REQUEST_CREATED” 时（即完成要号后），可通过上述 API 获取 “sorting_group” 字段值。

###**5. 基本发货流程**
####**5.1 发货 API 调用流程**

请注意，选择该物流选项的订单会自动拆分，使用以下 API 时，需确保在 request 中带入 “package_number” 字段：


3. 获取发货参数：调用 “v2.logistics.get_shipping_parameter” 或 “v2.logistics.get_mass_shipping_parameter”API


4. 执行发货要号：调用 “v2.logistics.ship_order” 或 “v2.logistics.mass_ship_order”API


5. 获取寄件编号：调用 “v2.logistics.get_tracking_number”API

####**5.2 获取虾皮出货标签（unpackage shipment label）流程**

按顺序调用以下 API，即可获取该包裹（package）对应的商品编码标签（unpackage shipment label）：

1. v2.logistics.get_shipping_document_parameter 
1. v2.logistics.create_shipping_document
1. v2.logistics.get_shipping_document_result
1. v2.logistics.download_shipping_document

该物流选项的订单仅支持 “THERMAL_UNPACKAGED_LABEL” 类型的 “shipping_document_type”

####**5.3 获取装箱单（TO label）流程**

按拣货分区（sorting group）完成装箱后，可通过以下 API 获取装箱单（TO label）文件：

- v2.logistics.download_to_label

请注意，每个装箱单（TO label）均为独立编码，需确保每个装箱单（TO label）的获取均通过独立调用该 API 实现；重复使用装箱单将影响交寄流程。

###**6. 其他流程介绍**
####**6.1 订单建立前获取虾皮出货标签（unpackage shipment label）**

我们支持在该物流选项的订单生成前，预先获取虾皮出货标签（unpackage shipment label），以便提前贴标备货；


可按顺序调用以下 API 获取文件：

1. v2.logistics.create_shipping_document_job：创建打印任务
1. v2.logistics.get_shipping_document_job_status：获取任务状态
1. v2.logistics.download_shipping_document_job：下载标签文件

请注意，商品必须已开启 “虾皮店到店 - 无包装隔日到” 物流选项，方可使用该流程调用 API。


*(*自2026/01/12 起，商店需选用 "虾皮商品编码" 出货标签模式，方可以使用此流程，若该商店为 "物流编号" 出货标签模式，使用此流程将无法取到可使用的标签)*

####**6.2 自定义虾皮出货标签（unpackage shipment label）**

在该物流选项中，我们推荐您可根据卖家操作习惯自行设计虾皮出货标签（unpackage shipment label）。您可参考本 FAQ 附加文件中 “自定义字段” 板块的规范，设计符合自身需求的标签。

- v2.logistics.get_shipping_document_data_info：使用 B2C 大物流的订单，可通过该 API 获取 “unpackaged_sku_id”及 “unpackaged_sku_id_qrcode”字段
- 规范概要（仅供参考，实际规范请以 Open Platform 网站上的 FAQ 文件为准）
 - 二维码 QR Code 生成规范
 - 禁用 ECI 模式
 - 禁止加入 UTF-8 或其他编码声明区段。
 - 自定义字段显示规范
 - 无字体大小及行数限制，以不影响非自定义区块显示为原则

###**8. 常见问题**

Q：该物流选项的订单会生成多个包裹，当多个包裹的物流状态不同时，订单状态会如何显示？


A：请参考本 FAQ 说明：https://open.shopee.com/faq/510


—-—-—-—-—-—-


Q：为什么通过 “v2.order.get_order_detail”API 获取订单详细信息时，外层的 “shipping_carrier”字段是空值？


A：选择该物流选项的订单会自动拆分，拆分后的订单在 “v2.order.get_order_detail”API 外层的 “shipping_carrier” 字段不会显示数值，请取用 “package_list”中各包裹（package）的 “shipping_carrier” 字段值。


—-—-—-—-—-—-


Q：为什么调用发货或获取虾皮出货标签的 API 时，系统返回错误信息？


A：选择该物流选项的订单会自动拆分，请先确认调用物流（logistics）相关 API 时，是否在请求参数（request）中包含 “package_number”字段。
