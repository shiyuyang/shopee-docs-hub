## 2.API 调用流程
### 2.1获取退货订单列表和详情

V2.returns.get_return_list API: 可获取一个店铺的退货退款申请列表，每个申请都将会返回一个 return_sn 作为唯一标识，买家针对同一个订单可能会提交多个 return_sn。返回参数中包含 order_sn 即为此退货退款申请关联的订单号。另外，接口支持筛选不同类型的退货退款申请，包括退货状态（status）、谈判状态（negotiation_status）、证据上传状态（seller_proof_status）、卖家赔偿状态（seller_compensation_status）。


V2.returns.get_return_detail API: 使用 return_sn 来获取退货退款申请详情。


`v2.returns.get_available_solutions` API: 允许卖家获得提供给买家的可用解决方案。

### 2.2 只退款

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=ZQS44uJfiLB2uePkmDcUyf%2FOScVqTwP44QMF%2FXAq8vCxcueM8wv8TRA1HeD%2Fpm%2Bsys7mddy3rKsIyozsVSnhwQ%3D%3D&image_type=png)


`v2.returns.confirm`：同意买家的退货申请，仅针对全部退款类型，买家无需退回商品，一旦同意，状态将会更新为 Accepted.

### 2.3 退货退款 （无争议）

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=90A5VuK3m2fGFkFQ%2FABNe%2FaDkoutBegqrE4Qs%2B68fOjgXC4VwjeANpouKZ%2BW%2BM5837%2FwDz7SO7IzAUTujvnBHA%3D%3D&image_type=png)


`v2.returns.offer` :卖家提供退货退款方案给买家选择。


`v2.returns.accept_offer`：卖家接受买家提供的退货退款方案。

### 2.4 退货退款 （有争议）

![图片](https://open.shopee.com/opservice/api/v1/image/download/?image_id=S0WmFtBKIV0Ta1Y59tSnmP4NCI%2BzTFMTpugSbD0uMoYSgCM%2B87IFDlEMv1xQSVINFq%2BlviNky340lF718EFjLA%3D%3D&image_type=png)


卖家可先调用 `v2.returns.get_available_solution`来确定该退货单的可选择的方案，之后调用`v2.returns.offer`r 接口给买家选择。


买家也会提供方案给卖家，如果卖家接受可调用`v2.returns.accept_offer`来接受 offer。如果不接受可调用`v2.returns.dispute`向争议中心提出争议。


目前 Open API 只支持**REQUESTED**和**PROCESSING**两种状态的退货退款申请发起 dispute。

#### 2.4.1 争议流程

卖家对买家提出的退货请求有争议，可至争议中心处理该请求。


`v2.returns.dispute`:用于卖家将退货订单升级至争议中心。


提出争议后卖家可以通过接口上传证据图片，目前不支持上传视频。


`v2.returns.convert_image`：转换图片，获取 Shopee 图片 URL


`v2.returns.upload_proof`:上传图片


`v2.returns.query_proof`:查询已上传的图片

### **退货退款申请类型**

（Return Refund Request Type）

- 0：普通退货退款（买家在收到包裹后发起的退货退款申请，申请依据为预计送达日期或 “已送达” 状态。）
- 1：运输途中退货退款（商品仍在运往买家的途中时，由买家发起的退货退款申请。）
- 2：当场拒收退货（买家在收货现场拒收包裹后，由配送员发起的退货退款申请。）
### **验证类型**

（Validation Type）

- seller_validation：适用于 “退货包裹需寄送至卖家处” 的退货退款申请：卖家需对包裹进行验证，并决定是否为买家退款，或发起争议。
- warehouse_validation：适用于 “退货包裹需寄送至仓库” 的退货退款申请：仓库需对包裹进行验证，并决定是否为买家退款，或发起争议。
### **逆向物流状态**

（Reverse Logistics Status）

#### **[普通退货场景]**
- LOGISTICS_PENDING_ARRANGE：退货申请当前处于 “待买家选择配送方式” 状态。无论是集成物流还是非集成物流，此状态含义一致。
- LOGISTICS_READY：买家已选择配送方式，等待系统创建物流请求。此时物流单号尚未生成。集成物流与非集成物流的此状态含义一致。
- LOGISTICS_REQUEST_CREATED：表示物流请求已成功创建，此时应可获取物流单号。
- LOGISTICS_PICKUP_RETRY：第三方物流公司将再次尝试从买家处揽收包裹。仅适用于集成物流（此状态由第三方物流公司同步至 Shopee 平台）。
- LOGISTICS_PICKUP_FAILED：第三方物流公司从买家处揽收包裹失败。仅适用于集成物流（此状态由第三方物流公司同步至 Shopee 平台）。
- LOGISTICS_PICKUP_DONE：
 - 集成物流：表示包裹已由第三方物流公司成功揽收；
 - 非集成物流：表示买家已提交发货凭证。
- LOGISTICS_DELIVERY_FAILED：包裹向卖家派送失败。仅适用于集成物流（此状态由第三方物流公司同步至 Shopee 平台）。
- LOGISTICS_LOST：包裹已被标记为 “丢失”。仅适用于集成物流（此状态由第三方物流公司同步至 Shopee 平台）。
- LOGISTICS_DELIVERY_DONE：包裹已成功派送至卖家。仅适用于集成物流（此状态由第三方物流公司同步至 Shopee 平台）。
#### **[运输途中退货场景]**
- Preparing（待处理）
- Delivered（已送达）
- Delivery Failed（派送失败）
- Lost（包裹丢失）
#### **[当场拒收退货场景]**
- Preparing（待处理）
- Delivered（已送达）
- Delivery Failed（派送失败）
- Lost（包裹丢失）
### **退货后物流状态**

（POST Return Logistics Status）


注：此状态仅适用于 “从仓库寄回给卖家的退货包裹”。

- POST_RETURN_LOGISTICS_REQUEST_CREATED：物流请求已成功生成，且可获取物流单号。
- POST_RETURN_LOGISTICS_REQUEST_CANCELED：仓库团队已取消该物流请求。
- POST_RETURN_LOGISTICS_PICKUP_FAILED：包裹揽收失败。
- POST_RETURN_LOGISTICS_PICKUP_RETRY：将再次尝试揽收包裹。
- POST_RETURN_LOGISTICS_PICKUP_DONE：包裹已成功揽收，正运往目的地。
- POST_RETURN_LOGISTICS_DELIVERY_FAILED：包裹派送失败，配送员将把包裹退回仓库。
- POST_RETURN_LOGISTICS_DELIVERY_DONE：包裹已成功派送至卖家。
- POST_RETURN_LOGISTICS_LOST：包裹已被标记为 “丢失”。
