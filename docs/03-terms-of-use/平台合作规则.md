## 1. 平台合作规则


平台合作规则文件强调了所有能够访问Shopee开放平台的开发者的预期行为。请注意，如果你不遵守我们的规则，可能会导致处罚。这些规则与我们的<u>数据保护政策</u>和<u>服务条款</u>相一致，这些政策和条款管理所有开放平台的开发者，确保Shopee继续成为我们所有合作伙伴和用户的一个公平和安全的平台。 



你应该熟悉我们政策的这四个关键领域。

1. 数据安全
1. 非法或不适当的行为
1. 共享信息的准确性
1. API性能和活动

## 2. 数据安全


为了确保属于卖家和Shopee平台的重要数据不被滥用或泄漏，所有使用Shopee开放平台的开发者都必须严格遵守我们的<u>数据保护政策</u>。 



在您访问Shopee开放平台之前，您需要接受并同意数据保护政策的条款。



Shopee对参与以下行为的开放平台开发者采取了严肃的态度：


1. 非法使用数据，如故意或无意地滥用、分享或泄露用户信息。
1. 非法收集用户数据，包括数据保护政策中规定的用途。
1. 分享或泄露你的APP的敏感信息，导致用户信息泄露。
1. 掩盖和利用Shopee开放平台上存在的缺陷或漏洞。
1. 隐瞒或不及时报告服务器资产，屏蔽您的平台，不接受Shopee开放平台的定期检查。


**💡 提示**



为了避免违反我们的数据保护政策，要积极主动地将数据泄露的风险降到最低。以下是你可以采取的一些预防措施：


- 对你的数据管理进行定期风险评估。
- 加强你的员工对数据安全最佳实践的认识。
- 与你的业务伙伴紧密合作，确保数据安全。
- 及时纠正Shopee通知的安全漏洞、安全风险、安全对策和其他相关要求。


欲了解更多信息，请详细阅读我们的<u>数据保护政策</u>。

## 3. 非法或不适当的行为


我们希望所有使用Shopee开放平台的开发者都能坚持公平的销售行为。任何被发现有损于我们平台的行为都将被严格处理。



Shopee对参与或被怀疑参与以下行为的商店或与之相关的开放平台用户采取了严肃的态度：


1. 使用网络爬虫来收集任何与Shopee有关的数据。
1. 履行欺诈，包括故意将空包裹/不相关物品运送给买家等行为。
1. 提供IP屏蔽等服务，帮助卖家避免被发现有违反Shopee卖家政策的行为。
1. 滥用内部API向卖家发送垃圾邮件。
1. 滥用聊天开放API，开发被禁止的功能，如主动更新订单、促销聊天广播和聊天机器人回复，或将自动消息功能伪装成手动消息，反之亦然。
1. 出售或交易卖方账户或假的当地商店
1. 提供或进行单向列表迁移，从Shopee的平台迁移到竞争对手的平台、网站或任何其他平台。
1. 所有刷屏活动，包括但不限于制造虚假订单、评论、粉丝和喜欢。
1. 任何其他不公平竞争的商业行为。


**💡提示**



为了避免因非法或不当行为而被标记，你应该始终保持诚实的销售做法。



欲了解更多信息，请详细阅读我们为卖家制定的<u>不公平竞争政策</u>。

## 4. 共享信息的准确性


透明度是良好工作关系的关键，这也是对所有获得Shopee开放平台访问权的开放平台开发者的期望。 



任何被发现故意隐藏商业信息的开放平台开发者都将受到惩罚。这包括但不限于提供不准确的细节，以及提供与提交描述不同的服务。



在您访问Shopee开放平台之前，您需要接受并同意<u>服务条款</u>。



**💡 提示**



为避免分享不准确的信息，请仔细检查您的开放平台申请。 



每当(i)您提供的服务(ii)您的IT资产或(iii)您的IT基础设施发生变化时，您也应在<u>Open Platform Console应用管理页面</u>上编辑您的APP详情。

## 5. API性能和活动


良好的API性能和活动可以确保我们的开发者为Shopee的用户提供高质量的服务。开放平台的开发者也应定期监测API调用成功率，并根据Shopee提出的要求及时做出必要的改变。



所有开发者都应满足以下要求：


1. 您的APP应在其创建后90天内上线并提供给用户。
1. 您的APP应在上线后的90天内进行至少1次API调用。
1. 您的APP需要有足够高的调用成功率，需保证每天90%以上的调用成功率。

**接口调用成功的判定标准为：HTTP 状态码为 200，且 response 中的 error 和 message 字段均为空*


***公共模块下的 API（例如 v2.public.refresh_access_token）不纳入计算范围*



Shopee平台将保留对不符合以上标准的开放平台开发者的处罚权，或对有上述违规行为的开发者的API调用不响应。



**💡 提示**



如果您的APP的API调用成功率持续过低，Shopee将通过您注册的电子邮件联系您。 为了保证信息畅通，请确保您的电子邮件正在使用中。如果您注册的电子邮件不          再使用，请通过发送邮件至openapi@support.shopee.com，更新您的电子邮件地址。

## 6. 第三方平台履行政策


第三方平台履行政策适用于所有帮助卖家从其他零售商或您的仓库直接向买家发货的第三方合作平台。 



制定该政策是为了确保提供履约服务的第三方合作平台能够帮助卖家及时、妥善地履行订单。



Shopee对有以下违规行为的第三方合作平台采取了严肃的态度。

**💡 提示**



了解如何<u>改善你的NFR和LSR</u>，或与你的零售商伙伴分享最佳做法，以避免履行处罚。



你应该始终保持低比例的预购清单和非整合订单，以确保买家及时得到他们的订单。

## 7. 处罚


如果出现违规行为，Shopee开放平台有权单方面调查开发者的行为，确定应该适用的规则，并根据违规行为的严重程度，对应用程序或开发者采取任何措施。 


有关处罚的详情如下：



**1. 警告通知 **

-  可通过电子邮件向开发者发出警告通知，并规定了整改的宽限期


**2.** **调用API 时对敏感数据进行屏蔽****（New）**

- 在调用以下 API 时，如 V2.order.get_order_detail、V2.logistics.get_shipping_document_data_info、V2.returns.get_return_detail、V2.order.get_buyer_invoice_info，将对买家的姓名、地址和联系方式等敏感数据进行屏蔽。


**3. 减少 API 调用限制****（New）**


Shopee 开放平台将确定一定比例的每日 API 限制减少。



**4. 暂停新店铺授权**


受处罚开发者名下的应用可能无法授权其应用访问新店铺/商家。在暂停前已获授权的店铺/商家可在授权过期前继续使用应用，但在暂停解除前可能不允许重新授权。



**5. 暂停应用程序**


受处罚开发者名下的应用可能无法进行 API 调用。应用程序的所有已授权店铺访问权限也可能被撤销。受处罚开发者可能无法授权其应用程序访问新店铺/商家。



**6. 删除 Shopee 开放平台账户**


受处罚开发者的开放平台账户可能被删除，并撤销所有权限。受处罚开发者名下的应用可能无法进行 API 调用。应用程序的所有已授权店铺访问权限也将被撤销。受处罚开发者可能不再能够授权其应用程序访问新店铺/商家。
